# 📝 Command: feedback-loop

> Defines how user feedback enters the development cycle.

## Template

### Feedback channels
- [ ] In-app feedback widget
- [ ] Support email
- [ ] Social media mentions
- [ ] App store reviews
- [ ] User interviews
- [ ] Analytics (implicit feedback)

### Triage process
1. All feedback goes to [tool/channel]
2. Categorized as: bug / feature request / UX issue / praise
3. Prioritized by: frequency × impact
4. Top items reviewed [weekly/biweekly]
5. Selected items added to roadmap

### Closing the loop
- Users who reported bugs get notified when fixed
- Feature requests get "considered" / "planned" / "shipped" status

# 📝 Definition of Done

> A task is "done" when ALL of these are true:

## Code
- [ ] Code is written and works as specified
- [ ] Code follows the project's naming and structure conventions
- [ ] No TODOs or hacks left in the code (or documented if intentional)

## Tests
- [ ] Unit tests pass
- [ ] New functionality has tests
- [ ] No existing tests broken

## Review
- [ ] Code review passed (or self-reviewed with checklist)
- [ ] Security considerations checked

## Documentation
- [ ] README updated if needed
- [ ] API docs updated if new/changed endpoints
- [ ] Comments for non-obvious logic

## Deployment
- [ ] Works in staging/preview
- [ ] No console errors or warnings
- [ ] Responsive on target devices
